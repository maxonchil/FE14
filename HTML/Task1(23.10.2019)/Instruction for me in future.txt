При вводе в консоль "node app.js" запускается сервер на порту (с конфигов которые записаны app.js), который указан в ".env" ( в этмо доке указывается чисто порт)
При вводе в консоль "npx lite-server" запускается сервер на порту 8009, с конфигов, которые записаны в bs-config.json
Получается, у тебя есть 2 способа запустить свой сервер. Один - используя lite-server, второй - используя express
В варианте, когда сервер стартует с файла app.js, есть возможность изменять заголовки http запросов ( потому что это такой код приписанный в файле )

Краткая инструкция по развёртыванию сервера(lite-server):
1. "npm init -y" в папке проекта
2.  Скачать пак Lite-server (npm lite-server) в папку проекта
3. Внести нужные изменения в файл package.json
4. Внести конфиги в файл bs-config.json и создать папку "public"в папке проекта
5. "Связать" файл конфигов с сервером командой в консоле "lite-server bs-config.json"
6. npx lite-server - запускает сервер

Краткая инструкция по развёртыванию сервера(express):
1. "npm init -y" в папке проекта
2. Скачать пак express (npm express) в папку проекта
3. Создать файл app.js в папке проекта
4. Внести необходимые изменения в файл package.json
5. Установить через консоль dotenv - что бы можно было создать файл с расширением .env
6. Прописать порт в .env
7. Внести необходимые изменения в файл app.js (задать конфиги)
8. Сервер запускается командой "node app.js" или "npm run dev"(при условии что в файле package.json прописан в объекте scripts - скрипт который ссылайется на файл app.js)
